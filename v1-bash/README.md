# kudig.sh v1.0 - Bash 版本

> ✅ **生产可用** - 稳定的 Bash 脚本实现

## 简介

`kudig.sh` v1.0 是一个高效的 Kubernetes 节点诊断日志分析工具，用于分析 `diagnose_k8s.sh` 收集的诊断数据。

## 核心特性

- **Bash 脚本实现**：轻量级、无依赖、开箱即用
- **离线分析模式**：分析 `diagnose_k8s.sh` 收集的诊断数据
- **43项异常检测规则**：涵盖系统资源、进程服务、网络、内核、容器运行时、Kubernetes 等维度
- **中英文双语报告**：支持文本和 JSON 两种输出格式
- **智能排查建议**：针对每个异常提供详细的排查步骤和解决方案
- **逐项详细输出**：显示每个检查项的名称、状态和结果

## 快速开始

### 1. 安装

```bash
# 直接使用（无需安装）
chmod +x kudig.sh
```

### 2. 收集诊断数据

```bash
# 在 Kubernetes 节点上收集诊断数据
sudo ./diagnose_k8s.sh
# 生成目录: /tmp/diagnose_1702468800
```

### 3. 分析诊断数据

```bash
# 基本使用
./kudig.sh /tmp/diagnose_1702468800

# 详细模式（显示每一步检查项）
./kudig.sh --verbose /tmp/diagnose_1702468800

# JSON 格式输出
./kudig.sh --json /tmp/diagnose_1702468800

# 保存报告到文件
./kudig.sh -o report.txt /tmp/diagnose_1702468800
```

## 命令行选项

| 选项 | 说明 |
|-----|------|
| `-h, --help` | 显示帮助信息 |
| `-v, --version` | 显示版本信息 |
| `--verbose` | 详细输出模式（显示每一步检查项） |
| `--json` | 输出 JSON 格式（默认文本格式） |
| `-o, --output <文件>` | 保存报告到指定文件 |
| `<诊断目录>` | diagnose_k8s.sh 生成的诊断目录路径 |

## 内置检测规则（43 项）

### 1. 系统资源检测（6 项）
- CPU 负载检测（2倍/4倍核心数阈值）
- 内存使用检测（85%/95%阈值）
- 磁盘空间检测（90%/95%阈值）
- 文件句柄检测（>50000警告）
- 进程/线程数检测（>5000/10000阈值）
- 磁盘使用检测

### 2. 进程与服务检测（6 项）
- Kubelet 服务检测
- 容器运行时检测（Docker/Containerd）
- ps 命令挂起检测
- D 状态进程检测
- Runc 进程检测
- Firewalld 检测

### 3. 网络检测（5 项）
- 连接追踪表检测（80%/95%阈值）
- 网卡状态检测
- 默认路由检测
- Kubelet 端口监听检测
- Iptables 规则检测

### 4. 内核检测（7 项）
- 内核 Panic 检测
- OOM Killer 检测
- Messages 日志 OOM 检测
- 文件系统只读检测
- 磁盘 IO 错误检测
- 内核模块加载检测
- NMI Watchdog 检测

### 5. 容器运行时检测（4 项）
- Docker 启动检测
- Docker 存储驱动检测
- Containerd 容器创建检测
- 镜像拉取检测

### 6. Kubernetes 组件检测（9 项）
- PLEG 状态检测
- CNI 插件检测
- Kubelet 证书检测
- API Server 连接检测
- Kubelet 认证检测
- Pod 驱逐检测
- 节点状态检测
- 磁盘压力检测
- 内存压力检测

### 7. 时间同步检测（1 项）
- NTP/Chrony 状态检测

### 8. 配置检测（5 项）
- Swap 配置检测
- IP 转发检测
- bridge-nf-call-iptables 检测
- ulimit open files 检测
- SELinux 检测
